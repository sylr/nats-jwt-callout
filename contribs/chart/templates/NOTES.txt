Applying config changes:
{{- if .Values.restartOnConfigChange }}
A checksum annotation triggers a rolling restart whenever the rendered config or
policy changes, so `helm upgrade` rolls the pods automatically.
{{- else }}
restartOnConfigChange is disabled; roll the pods yourself after a config change:

  kubectl --namespace {{ .Release.Namespace }} rollout restart deployment/{{ include "nats-oidc-callout.fullname" . }}
{{- end }}

The binary also hot-reloads on SIGHUP, but the distroless image has no shell, so
you cannot `kubectl exec ... kill`. For zero-restart reloads, use a controller
that signals PID 1 (e.g. stakater/reloader with a signal strategy).
